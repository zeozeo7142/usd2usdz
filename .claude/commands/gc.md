마지막 git commit 이후 변경된 내용을 요약하여 커밋하고, 가능하면 push까지 수행한다.

## 작업 순서

1. 아래 명령어로 현재 상태를 파악한다.
   ```bash
   git status
   git diff HEAD
   git log --oneline -5
   ```

2. 변경 내용을 분석하여 한국어로 간결한 커밋 메시지를 작성한다.
   - 형식: `[타입] 변경 내용 요약`
   - 타입: `추가`, `수정`, `삭제`, `리팩토링`, `문서`, `설정` 중 적절한 것 선택
   - 여러 변경이 있으면 가장 중요한 것을 제목으로, 나머지는 본문에 간단히 나열
   - 50자 이내로 간결하게

3. 변경된 파일을 스테이징하고 커밋한다.
   ```bash
   git add [변경된 파일들]
   git commit -m "[작성한 커밋 메시지]"
   ```

4. 원격 저장소가 설정되어 있으면 push를 시도한다.
   ```bash
   git remote -v        # 원격 저장소 확인
   git push             # push 시도
   ```
   - push 성공 시: 결과 보고
   - push 실패 시: 실패 이유를 설명하고 사용자에게 안내

## 주의사항

- 민감한 파일(.env, 비밀키, 개인정보 등)이 포함되어 있으면 스테이징하지 않고 사용자에게 알린다.
- 커밋 전 스테이징할 파일 목록을 사용자에게 보여준다.
- force push는 절대 하지 않는다.
