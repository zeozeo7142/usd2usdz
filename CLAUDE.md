# CLAUDE.md — 프로젝트 작업 지침

## 라이선스 사전 검토 (필수)

**라이브러리, 패키지, 도구를 설치하기 전에 반드시 라이선스를 확인하고 사용자에게 보고한 뒤 승인을 받아야 한다.**

### 확인 항목

| 항목 | 확인 내용 |
|------|-----------|
| 라이선스명 (SPDX) | MIT / Apache-2.0 / GPL-3.0 / TOST-1.0 등 |
| 상업적 사용 | 허용 여부 |
| 특허 조항 | 특허 소송 시 라이선스 종료 조항 존재 여부 |
| 소스 공개 의무 | Copyleft(GPL 계열) 여부 |
| 배포 제한 | 수정본 배포 시 동일 라이선스 적용 여부 |

### 보고 형식 (설치 전 사용자에게 제시)

```
패키지명: foo
라이선스: Apache-2.0
상업적 사용: 허용
특허 조항: 있음 (특허 소송 시 라이선스 종료)
기타 제한: 없음
→ 설치해도 될까요?
```

### 즉시 설치 불가 (반드시 확인 필요)

- GPL / LGPL / AGPL 계열: 소스 공개 의무 발생 가능
- 상업적 사용 금지 조항 포함 라이선스
- 사용자별 라이선스 계약(EULA) 필요 패키지
- 라이선스 불명확 또는 미공개 패키지

## 패키지 설치 일반 원칙

- `pip install`, `apt install`, `conda install`, `npm install`, `docker pull` 등 **모든 형태의 설치는 사전 승인 필수**
- 승인 없이 설치 금지. 명시적 "응", "ok", "해줘" 등의 응답을 받은 후에만 실행
- 설치 후에는 실제 설치된 버전을 확인하고 보고

---

## 문서 작성 지침

- **TODO.md**: 성공한 경로만, 재현 가능한 체크리스트 형식 → `cTODO.md` 참고
- **TM.md**: 성공·실패 모두 포함, 이유-작업-결과 서술 형식 → `cTM.md` 참고
- 문서에 명령어를 기재할 때는 실제 실행한 경로와 값을 그대로 사용

---

## 파일/디렉토리 명명 규칙

- 출력 디렉토리: 입력 소스 디렉토리명을 따른다 (예: 입력이 `USDZ_ETRI1`이면 출력도 `USDZ_ETRI1`)
- 파일명에 공백 포함 금지 (공백 → 언더스코어)
- 중간 산출물과 최종 결과물을 구분할 수 있는 이름 사용

## 기타

- 파괴적 작업(파일 삭제, 덮어쓰기, git reset 등)은 실행 전 반드시 확인
- 동일 파일명으로 덮어쓸 경우 사전 고지
